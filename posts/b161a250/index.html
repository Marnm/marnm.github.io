使用服务搭建Hexo个人博客

使用服务器搭建Hexo个人博客详细

在学习的过程中，被动接受知识对你的提高是轻微的；
只有主动学习，主动思考，才能学习的透彻，至少是高于被动学习吧。
搭建个人博客，最大化的利用你所学的知识，检验是否学习的透彻。

开始搭建个人博客

这里我使用的配置是本地机器Win 10_x64和腾讯云服务器CentOS 6.5_x64

配置本地环境

安装Node.js和Git，安装时默认一直下一步就好。


设置Git用户名和邮箱

git config –list

git config –global user.name “用户名”
git config –global user.email “邮箱地址”
或修改用户名和邮箱
git config –global –replace-all user.name “用户名”
git config –global –replace-all user.email “邮箱地址”

安装Hexo

npm install -g hexo-cli

配置Hexo站点目录

mkdir blog // 创建存放Hexo站点的文件夹
hexo init blog // 生成Hexo站点
hexo s –debug // 站点测试, 在浏览器中打开http://localhost:4000


生成SSH密钥

在本地生成一对SSH密钥，用于与服务器连接
ssh-keygen -t rsa -C “echo@qq.com“

#第一个提示输入保存文件名，默认为空，回车。

#第二个提示输入密钥，默认为空，回车

#第三个确认刚输入密钥，默认为空，回车

配置服务器环境

安装Node.js

#yum install -y nodejs

安装Git

由于Redhat Linux，Oracle Linux，CentOS，Scientific Linux等yum库的Git版本太低，这里不推荐使用，我们直接去下载最新版源码包编译安转。
RHEL和衍生产品通常都会发布旧版本的git。您可以从源代码下载tarball和构建，或使用第三方存储库（如IUS社区项目）来获取更新版本的git。

首先安转Git依赖包

#yum install curl-devel expat-devel gettext-devel openssl-devel zlib-devel

#yum install gcc perl-ExtUtils-MakeMaker

#yum remove git // 如果之前有安转旧版本Git先卸载

下载解压Git源码包
mkdir /usr/local/git
cd /usr/local/git
wget https://mirrors.edge.kernel.org/pub/software/scm/git/git-2.9.5.tar.gz
tar -zxvf git-2.9.5.tar.gz

编译安装
cd git-2.9.5
make prefix=/usr/local/git all
make prefix=/usr/local/git install
echo “export PATH=$PATH:/usr/local/git/bin” >> /etc/bashrc //#添加环境变量
source /etc/bashrc ##使配置立即生效

设置Git默认路径

#ln -s /usr/local/git/bin/git-upload-pack /usr/bin/git-upload-pack

#ln -s /usr/local/git/bin/git-receive-pack /usr/bin/git-receive-pack

创建git用户组和用户,用来运行Git服务

#groupadd git

#useradd git -g git

#passwd git
切换到git用户

#su - git

设置用户权限

##git用户目录下:

#mkdir .ssh ##新建文件夹

#chmod 700 .ssh

#touch .ssh/authorized_keys ##新建文件

#chmod 600 .ssh/authorized_keys
cd /home
chown -R git:git git设置git文件夹所有者，去除无关用户的权限

修身ssh配置文件，开启RSA认证

#vi /etc/ssh/sshd_config

#找到如下三个选线，修改
1.RSAAuthentication yes
2.PubkeyAuthentication yes
3.AuthorizedKeysFile home/git/.ssh/authorized_keys

保存:wq
回到本地机器，将刚才生成的SSH密钥导出上传到服务器

scp .ssh/id_rsa.pub root@192.168.1.2:/home/git/.ssh/authorized_keys
我的win10系统是自带有scp命令的，win7可能没有，可以打开git bash命令行，git工具有，或者安装putty，可以使用pscp

部署hexo博客

初始化git仓库

#cd /home/git

#git init –bare blog.git

#chown -R git:git blog.git ##每生成一个git仓库都要修改用户权限

克隆仓库到本地，测试本地机器与服务器的连通

git clone git@your-ip:/home/git/blog.git

…
warning: You appear to have cloned an empty repository.
…

在web目录下clone

这里我直接使用Apache的默认路径，也可以修改为别的目录，例如：mkdir /var/www/hexo-blog

git clone /home/git/blog.gig /var/www/html
chown -R git:git /var/www/html

设置Git Hook

使用Git Hook把Hexo生成的静态web资源自动部署到web目录下
vim /home/git/hooks/post-receive
在DeployPath=””中填入你的web目录

#!/bin/bash
#author: @illgo
#author blog: http://www.illgo.cn/

#判断是不是远端仓库

IS_BARE=$(git rev-parse --is-bare-repository)
if [ -z "$IS_BARE" ]; then
echo >&2 "fatal: post-receive: IS_NOT_BARE"
exit 1
fi

unset GIT_DIR
DeployPath="/var/www/html/"

echo "==============================================="
cd $DeployPath
echo "deploying the myblog web"

#git stash

#git pull origin master

git fetch --all
git reset --hard origin/master


time=`date`
echo "web server pull at webserver at time: $time."
echo "================================================"

配置Git部署

回到本地，切换到你生成的hexo站点目录
cd blog
npm install hexo-deployer-git
安装hexo的Git部署依赖包

修改hexo站点配置文件_config.yml, 拉到最底部

type: git
repo: git@your-ip:/home/git/blog.git

hexo clean && hexo g && hexo d

最后

出于安全考虑，修改一下git用户权限,禁止shell登录

找到git用户信息
vim /etc/passwd

git:x:502:502::/home/git:/bin/bash
## 改为
git:x:502:502::/home/git:/usr/local/git/bin/git-shell
